# 🏫 Modelagem de Sala de Aula, Turma e Curso (POO)

Aplicação desenvolvida como exercício de Programação Orientada a Objetos, focada na modelagem de relacionamentos entre classes, encapsulamento e organização de responsabilidades.

---

## 🧠 Sobre

Este projeto foi desenvolvido durante o curso técnico no IFSul como um exercício de Programação Orientada a Objetos (POO).

A atividade fazia parte do processo de introdução aos princípios SOLID, com foco na separação de responsabilidades, encapsulamento e organização do código.

O objetivo era modelar uma estrutura acadêmica simples, relacionando entidades como alunos, turma, sala de aula e curso, com cada classe possuindo sua própria responsabilidade.

---

## ⚙️ Estrutura

O sistema foi dividido em classes com responsabilidades específicas:

- **Aluno** → representa o aluno e seus dados  
- **Curso** → representa o curso relacionado à turma  
- **Turma** → contém uma lista de alunos e um curso  
- **SalaDeAula** → contém uma lista de alunos  
- **Main** → responsável por executar e testar a estrutura criada  

---

## ⚙️ Conceitos trabalhados

- Relacionamento entre classes  
- Encapsulamento  
- Uso de getters e setters  
- Separação de responsabilidades  
- Organização de código orientado a objetos
